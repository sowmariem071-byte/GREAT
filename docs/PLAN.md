# 内容排期系统开发计划文档

本计划文档按照 [开发需求文档](./REQUIREMENTS.md) 执行。需求文档是主线，计划文档负责把需求拆成阶段、任务顺序、验收重点和下一步优先级。每次执行计划中的任务后，必须同步更新 [开发进度文档](./PROGRESS.md)。

## 执行规则

1. 开发前先确认本次任务是否改变需求主线。
2. 需求有变化时，先更新 `docs/REQUIREMENTS.md`。
3. 需求影响阶段、优先级或验收时，更新 `docs/PLAN.md`。
4. 完成代码、样式、数据库、交互或文档变更后，追加 `docs/PROGRESS.md`。
5. 每个阶段完成时至少执行对应验证命令，并记录验证结果。

## 统一验收命令

常规代码变更后优先执行：

```powershell
npx prisma validate
npm run typecheck
npm run lint
npm run build
```

纯文档变更至少执行文档结构检查，例如：

```powershell
rg -n "开发需求文档|开发计划文档|开发进度文档" docs
```

如果本地开发服务器因为构建占用 `.next`，先停止 3000 端口监听，再运行构建，完成后重启 `npm run dev`。

## 需求变化处理流程

当用户提出新需求或改变原有需求时，必须按以下顺序处理：

1. 判断这是新增需求、修改需求、删除需求还是视觉偏好调整。
2. 更新 `docs/REQUIREMENTS.md` 中对应章节。
3. 如果影响后续开发顺序，更新 `docs/PLAN.md` 的阶段状态或下一步优先级。
4. 实施代码或样式修改。
5. 运行对应验证。
6. 追加 `docs/PROGRESS.md`，记录需求变化、实现内容、影响文件和验证结果。
