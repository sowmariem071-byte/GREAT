# 内容排期系统文档索引

本目录用于沉淀内容排期系统的需求主线、开发计划、开发规范、进度记录和模块设计。后续任何代码、页面、交互、数据结构或业务规则变化，都必须同步到对应文档。

## 文档职责

- [开发需求文档](./REQUIREMENTS.md)：项目需求主线，记录已经确认的角色、流程、页面、权限、业务规则和视觉原则。需求变化先改这里。
- [开发计划文档](./PLAN.md)：按需求主线拆分阶段、优先级、验收标准和下一步执行顺序。计划变化跟随需求变化更新。
- [开发协作规范](./DEVELOPMENT.md)：记录开发流程、文档同步规则、验证要求和本项目固定约定。
- [开发进度文档](./PROGRESS.md)：记录每一次真实变更，包含目标、变更、影响文件、验证和下一步。
- [配色方案模块设计](./THEME_CUSTOMIZATION.md)：用户可调整配色的模块方案、数据结构和落地步骤。

## 同步规则

1. 需求新增、删减或调整时，先更新 [开发需求文档](./REQUIREMENTS.md)。
2. 如果需求影响执行顺序、模块优先级或验收标准，同步更新 [开发计划文档](./PLAN.md)。
3. 每次实际开发、样式调整、交互调整、数据库调整或文档调整后，追加记录到 [开发进度文档](./PROGRESS.md)。
4. [开发进度文档](./PROGRESS.md) 不替代需求文档，需求结论必须回写到 [开发需求文档](./REQUIREMENTS.md)。
