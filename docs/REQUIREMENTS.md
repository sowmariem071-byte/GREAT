# 内容排期系统开发需求文档（主线）

本文档是内容排期系统的需求主线。后续任何需求变化，都必须先同步到本文档，再同步到 [开发计划文档](./PLAN.md) 和 [开发进度文档](./PROGRESS.md)。

## 1. 产品目标

内容排期系统用于帮助内容团队每天掌握脚本、视频剪辑、审核、可发布、排期、发布和库存情况，让管理员、编导、剪辑都能在自己的工作台里看到今日目标、当前进度、待处理事项和风险。

系统第一版目标是做成真实可用的内部工作台，而不是静态展示页。核心原则是：流程清楚、页面简洁、状态可追踪、任务可闭环、团队工作量一眼能看懂。

| 编导 | 王煊 | 3 条中式育儿 + 1 条女性 | 创建脚本、下达视频任务、审核成片、安排发布、查看个人进度 |
| 编导 | 单萱萱 | 3 条中式育儿 | 创建脚本、下达视频任务、审核成片、安排发布、查看个人进度 |

管理员必须可以新增、停用、删除或调整人员信息，并修改每个人的每日目标。

## 3. 核心业务流程

主流程为：

```text
选题 / 素材 / 脚本 -> 剪辑 -> 审核 -> 可发布 -> 排期 -> 发布 -> 库存
```

实际状态流转为：

```text
脚本入库 -> 下达视频任务 -> 待剪辑 -> 剪辑中 -> 待审核 -> 需修改 -> 待审核 -> 审核通过 -> 可发布 / 屯片 -> 已排期 -> 已发布 -> 库存归档
```

审核可以多轮循环。每一轮审核都必须保留历史，包括视频链接、审核结论、修改文字、截图附件、提交人和提交时间。

## 4. 内容分类规则

脚本原始类别有三种：

- 传统
- 协助
- 女性

其中：

- 传统和协助都计入中式育儿脚本。
- 女性计入女性脚本。
- 视频任务展示层面使用两类：中式育儿、女性。

## 5. 核心页面需求

### 登录页

- 使用账号密码登录。
- 初始账号使用姓名拼音首字母缩写：龚锐 `gr`、胡欣怡 `hxy`、王煊 `wx`、单萱萱 `sxx`、贺玲玥 `hly`、黄炜琪 `hwq`。
- 初始账号统一密码为 `zzb888`。
- 登录后根据角色进入对应工作台。
- 使用数据库会话 Cookie。

### 个人主页 / 工作台

每个角色都要有不同主页。主页简约为主，需要头像或个人空间感，重点展示本岗位今日工作进度。

编导工作台当前确认样式：

- 背景使用暖灰白，不使用大面积红橙渐变。
- 主内容区居中，最大宽度控制在 1280px 到 1360px。
- 顶部标题为“编导工作台”。
- 副标题为“今日脚本、剪辑流转与发布库存一屏掌握”。
- 右侧展示今日完成率、百分比和完成数。
- 核心数据卡片包括今日脚本目标、待审核、剪辑中、发布库存。
- 下方包括“今日目标”和“当前待处理”两块。
- 删除会破坏高级感的右侧悬浮工具按钮、右下角装饰物和过满信息。

管理员工作台需要展示：

- 管理员首页位于 `/dashboard`，使用“管理员工作台 / 管理员 WORK SIGNAL”作为顶部主标题区。
- 视觉结构与编导、剪辑首页保持同一套暖灰白高级工作台语言。
- 顶部右侧展示全员完成率、百分比和完成数。
- 顶部核心数据卡包括今日总进度、脚本产出、剪辑交付、审核风险，并且可点击进入对应详情页。
- 全员今日目标和完成情况。
- 全员今日进度以人员卡片展示，卡片需要显示姓名、角色、今日目标完成数和细进度条，并可进入人员详情。
- 今日脚本、视频产出、剪辑交付、审核、发布和库存概览。
- 按 70s 内、70s-130s、130s+ 统计每天完成的不同时长视频条数。
- 右侧需要展示“风险与待处理”，包含逾期任务和待审核任务，并可进入视频任务池对应筛选。
- 右侧需要展示“发布与时长”，包含可发布、屯片池和今日交付时长结构。

剪辑工作台需要展示：

- 今日剪辑完成情况。
- 待剪辑、剪辑中、需修改、已交付、今日完成、逾期任务。
- 按 70s 内、70s-130s、130s+ 统计每天完成的视频条数。
- 剪辑首页位于 `/dashboard`，必须参考编导首页的确认结构，而不是继续使用旧版通用岗位进度页。
- 剪辑首页使用“剪辑工作台 / 剪辑 WORK SIGNAL”作为顶部主标题区，展示今日完成率和 `已交付 / 目标`。
- 剪辑首页顶部核心数据卡包括今日剪辑目标、剪辑流转、修改返工、待审核。
- 剪辑首页顶部核心数据卡必须可点击跳转：今日剪辑目标进入 `/editing` 总队列，剪辑流转进入 `/editing?queue=flow` 只看待剪辑和剪辑中，修改返工进入 `/editing?status=NEEDS_REVISION`，待审核进入 `/editing?status=PENDING_REVIEW`。
- 剪辑首页下方左侧展示今日目标和 70s 内、70s-130s、130s+ 今日交付结构；右侧展示当前最优先处理任务。
- `/editing` 是剪辑任务处理页，使用“剪辑工作台 / EDITOR WORK SIGNAL”任务视图，展示状态筛选和任务卡片，视觉重点必须聚焦“我的剪辑队列”。
- `/editing` 顶部核心信号卡包括待剪辑、剪辑中、需修改、待审核，点击可筛选对应任务。
- `/editing` 不展示“今日负载”和“今日时长结构”侧栏；时长结构统计只放在管理员或全局统计视角中。
- `/editing` 剪辑队列里的任务卡片必须可点击打开完整视频详情，详情包括任务信息、脚本详情、素材链接、成片链接、编导、剪辑、计划交付、计划发布、交付/审核/发布时间等字段。
- 剪辑侧视频详情里待审核状态只展示等待和详情，不提供编导审核建议填写入口。
- 剪辑任务的状态更新、视频链接和秒数填写必须通过弹窗完成。
- 待审核任务在剪辑侧只显示等待编导审核，不提供重复提交入口。

### 脚本资产库

- 编导每天上传自己撰写的文案。
- 新增脚本时填写标题、正文、类别。
- 脚本创建后进入脚本库。
- 脚本库以卡片形式展示，不使用普通表格作为主视觉。
- 点击脚本卡片可以打开详情弹窗。
- 详情弹窗可以查看并修改标题、类别和正文。
- 管理员可以修改全部脚本，编导只能修改自己的脚本。
- 弹窗需要淡入、上浮、遮罩模糊等轻量动效，避免突兀。
- 左侧可以挂“今日进度”，右侧可以挂“未来 7 天发布日历”和“待安排”信息。

### 视频任务池

- 编导可以下达剪辑任务。
- 下达任务时必须从脚本库中选择脚本，脚本选择必须以卡片形式展示。
- 如果没有找到对应脚本，需要提供“创作新脚本并下发”的入口。
- 新增视频任务必填项：脚本、素材链接、计划交付日期。
- 可选项：计划发布日期、发布时间、备注。
- 剪辑人自动分配给贺玲玥或黄炜琪，规则为“未完成任务最少”。
- 视频任务流需要使用列表形式展示，每行展示创建日期、标题、类型（中式育儿/女性）、剪辑人、视频秒数、状态、计划交付、审核结果、可发布状态。
- 点击视频任务列表行可以打开详情弹窗。
- 详情弹窗展示完整任务信息，包括任务说明、素材链接和成片链接。
- 详情弹窗内“查看脚本详情”需要额外弹出独立脚本详情窗口，专门展示当前任务绑定的脚本信息，不能把脚本正文直接塞在视频详情底部。
- 编导和剪辑姓名在视频详情中仅作为普通任务信息展示，不提供人员主页跳转。
- 状态为“已发布”时，状态标签可跳转到已发布日历规划池。
- 状态为“待审核”时，状态标签可直接打开审核建议填写窗口。

视频任务池当前确认交互：

- 点击“任务总数”统计卡，进入全部视频任务流，清空队列和状态筛选。
- 待剪辑、剪辑中、待审核、需修改、可发布、屯片等状态数字必须可点击。
- 点击“剪辑流转”统计卡，进入剪辑流转任务视图，只看待剪辑和剪辑中。
- 点击“审核队列”统计卡，进入审核队列任务视图，只看待审核和需修改。
- 分组视图仍保留完整任务详情弹窗能力。
- 视频任务池主内容不展示左侧 Flow Completion、剪辑负载和排期信号侧栏；视频任务流模块需要横向扩展，占满原侧栏空间，保持简洁布局。

### 剪辑任务页

- 剪辑只能看到分配给自己的任务。
- 剪辑可以把任务从待剪辑更新为剪辑中。
- 剪辑完成后提交成片链接，并把状态更新为待审核。
- 如果编导要求修改，任务进入需修改。
- 剪辑修改完成后重新提交待审核。
- 审核通过后剪辑侧显示已交付。
- 页面需要支持按状态筛选。

### 审核中心

- 编导可以查看待审核视频。
- 审核中心位于 `/review`，需要使用“审核中心 / REVIEW SIGNAL”工作台结构。
- 顶部展示待审核、需修改、近期通过、近期退修四个审核信号。
- 主体区域必须使用列表形式展示视频，列表项只显示视频标题，方便视频数量多时快速浏览。
- 点击视频标题后，必须弹出该视频的完整审核详情窗口。
- 审核详情弹窗展示成片链接、素材链接、计划交付、计划发布、发布时间、秒数、脚本正文、上一轮反馈和本视频审核轮次。
- 打开审核时需要显示视频链接。
- 审核建议填写区域需要出现在弹窗里，支持文字修改建议和截图上传。
- 审核结论包括通过和需修改。
- 需修改时任务回到剪辑侧的需修改状态。
- 通过后进入可发布或屯片逻辑。
- 审核历史必须完整保留。
- 最近审核记录需要在页面下方展示，方便回看通过和退修情况。

### 排期日历

- 审核通过且有计划发布日期的视频进入可发布或排期。
- 排期日历位于 `/schedule`，需要使用“排期日历 / PUBLISH PLAN”工作台结构。
- 顶部展示已排期、今日发布、待安排、已发布四个排期信号。
- 页面需要突出未来一周发布日历，按今天起连续 7 天展示每日计划。
- 页面需要明确展示“已规划到 XX/XX XX:XX，真棒”或暂无排期时的待安排提示。
- 右侧需要展示待安排池，包含可发布、屯片或已审核但尚未排期的视频。
- 支持查看计划发布日期、发布时间和发布状态。
- 支持手动标记已发布，并填写发布链接。
- 已发布内容进入库存归档视图。
- 已发布归档区位于排期日历页，并需要支持从视频详情状态标签定位。

### 库存池

- 没有计划发布日期且审核通过的视频进入屯片池。
- 有计划发布日期或可发布的视频进入可发布列表。
- 已发布内容进入已发布库存。
- 库存池位于 `/inventory`，使用“库存池 / INVENTORY SIGNAL”工作台结构。
- 顶部信号卡展示可发布、屯片池、待发布、已发布四类库存状态。
- 页面主体需要分区展示可发布池和屯片池，帮助编导和管理员快速判断当前可调度内容。
- 库存页右侧不展示库存结构或已发布归档侧栏；原“最近发布”位置改为“去排期日历”入口。
- 页面需要支持可发布、已排期、屯片等状态入口过滤，并保留 `#stock`、`#ready` 等库存定位入口；已发布入口跳转到 `/schedule#published`。
- 库存页不展示剪辑时长结构，70s 内、70s-130s、130s+ 统计归入管理员或全局统计视角。

### 人员与目标设置

- 管理员可以新增人员、调整角色、停用人员、调整每日目标。
- 每个人员需要有头像、姓名、角色、状态、目标配置。
- 人员详情页需要展示该人员的岗位工作情况。
- 人员与目标设置页位于 `/people`，使用“人员与目标设置 / TEAM TARGETS”工作台结构。
- 顶部信号卡展示团队成员、编导目标、剪辑目标、待处理负载。
- 成员以卡片形式展示头像、姓名、角色、账号、启用状态、今日完成、细进度条和当前目标配置。
- 新增成员、调整目标、启用或停用账号等维护动作优先通过弹窗或轻量按钮完成，不在页面铺开长表单。
- 管理员自己的账号在界面上不允许直接停用，避免误操作导致当前管理员失去访问入口。
- 人员详情页仅允许管理员查看任意成员；非管理员只能查看自己的人员详情，访问其他成员详情时回到自己的首页。

### 个人设置与配色

- 用户可以自定义头像。
- 用户可以调整配色方案。
- 配色优先级：用户自定义主题 > 角色默认主题。
- 配色模块保留浅色高级后台基调，不能破坏整体一致性。
- 个人设置页位于 `/settings`，使用“个人设置 / PERSONAL SPACE”工作台结构。
- 页面顶部需要展示用户头像、姓名、账号、角色和头像更换入口，强化个人空间感。
- 所有角色的个人设置页都必须在退出登录左侧显示“修改头像”按钮，不能因为管理员、编导或剪辑身份不同而隐藏文案。
- 个人设置页保持简洁布局，不展示顶部概览信号卡、右侧头像预览卡或配色规则说明卡。
- 配色设置需要包含实时视觉预览、主题预设卡片和自定义配色弹窗。
- 个人配色只影响强调色、辅助色和柔和底色，不改变全局暖灰白背景、深灰黑文字和轻玻璃卡片基调。
- 用户点击应用配色后，所有主要页面的导航、按钮、标签、重点数字、进度条和工作台点缀色都必须使用用户主题，不允许继续使用写死的单一赤陶色。

## 6. 关键业务规则

- 新增视频任务时，剪辑人按未完成任务最少自动分配。
- 未完成任务包括待剪辑、剪辑中、待审核、需修改等未闭环状态。
- 审核通过后，如果没有计划发布日期，则进入屯片池。
- 审核通过后，如果有计划发布日期，则进入可发布或排期。
- 发布第一版为手动标记，不对接外部平台。
- 视频成片第一版只保存链接，不上传成片文件。
- 头像和审核截图使用 S3 兼容对象存储，数据库只保存文件元数据和访问地址。
- 每天 16:30 按 Asia/Shanghai 时区生成未完成提醒。
- 第一版提醒只做站内提醒和页面标红。
- 服务端业务动作必须校验角色、数据归属和状态流转，不能只依赖页面按钮是否显示。
- 剪辑只能更新仍处于待剪辑、剪辑中或需修改的视频任务，不能修改待审核、已排期、已发布等闭环任务。
- 审核只能针对待审核视频提交结论，不能绕过剪辑提交直接审核其他状态的视频。
- 排期只能安排已审核通过、可发布、屯片或已排期调整中的视频，不能排期仍在剪辑或待审核的视频。
- 手动标记已发布只能针对已排期视频执行，不能重复发布或跳过排期状态。
- 管理员账号停用动作必须防止当前管理员停用自己。

## 7. 数据模型主线

当前核心数据模型包括：

- `User`：姓名、角色、头像、状态、密码哈希。
- `Session`：登录会话。
- `DailyTarget`：用户每日目标，中式育儿、女性、剪辑数量。
- `Script`：标题、正文、类别、作者、入库日期。
- `VideoTask`：脚本、素材链接、计划交付、计划发布、发布时间、剪辑人、秒数、状态、备注。
- `ReviewRound`：视频链接、审核结论、修改文字、轮次、提交人、时间。
- `ReviewAttachment`：审核截图对象存储地址。
- `PublishSchedule`：计划发布日期、发布时间、发布状态、发布链接。
- `Notification`：站内提醒和未读状态。
- `ThemePreference`：用户配色配置。

## 8. 技术与部署要求

- 使用 Next.js + TypeScript。
- 使用 Prisma 管理 PostgreSQL。
- 使用 S3 兼容对象存储保存头像和审核截图。
- 支持 Docker 服务器部署。
- 支持使用 Supabase 托管 PostgreSQL 作为公网数据库。
- 支持使用 Vercel 部署公网 Web 服务。
- 生产环境必须通过 Vercel 环境变量配置 `DATABASE_URL`、`SESSION_SECRET`、`APP_URL` 和对象存储相关密钥，禁止提交 `.env` 或明文密钥。
- Supabase 第一版只作为 Prisma 直连数据库使用，不依赖 Supabase Data API；数据库表启用 RLS 且不开放匿名客户端策略。
- 本地没有 Docker 时，需要保留可替代的开发配置和说明。
- 时区固定为 Asia/Shanghai。

## 9. UI 与交互原则

- 整体风格高级、大气、简约、有留白，适合年轻创意团队日常使用。
- 当前正式视觉基调为暖灰白背景、白色或半透明玻璃卡片、深灰黑主文字、浅灰辅助文字、赤陶红或焦糖橙点缀。
- 页面不能塞满，留白要围绕内容模块服务。
- 需要填写的新增、下达、审核、排期、发布、人员目标和设置操作，优先用弹出窗口承载。
- 弹窗要轻量、丝滑、聚焦，避免突然弹出。
- 数据卡片不能像普通后台表格，要更像年轻创意团队的工作台。
- 全站卡片默认只保留标题、数字、状态和必要操作，不展示解释型小字、辅助说明或重复描述，避免画面信息量过多。
- 页面标题区和模块标题区默认不展示解释性副标题或说明段落，只保留必要标题、状态标签和操作入口。
- 退出登录入口统一只出现在个人设置页，不在首页、工作台或其他业务页面右上角展示。
- 不使用大面积红橙背景，不做游戏感，不做大屏炫技风。
- 主页面重点只保留与岗位有关的工作进度。
- 登录页需要作为正式入口设计，保持浅色高级基调，同时允许加入科技感动态背景元素、轻量轨道动效和工作流信号装饰，让入口不空洞但不影响登录聚焦。
- 登录页不展示默认账号、默认密码、种子账号列表或具体人员姓名，避免公网入口暴露账号提示。
- 登录页当前采用轻量顶部品牌栏加高级简约双栏布局：页面背景纯白，左上角显示“母婴事业部”，右上角显示“进入【蒸蒸爸】官网”并跳转到 `https://ihmzoectzzq0.meoo.fun/`；主体距离顶部品牌栏约 28px，主体宽度由视口宽度和视口高度共同决定，不再使用 `max-width: 1120px` 的小型居中容器；桌面端左右边距约 16px 到 24px，双栏 `1fr 1fr`，间距约 10px。
- 登录页左侧视觉卡和右侧登录卡在桌面端必须尺寸一致并接近正方形，使用视口宽高共同约束主体宽度，避免 1920px 宽屏下卡片只有约 554px 或底部出现大面积空白；左侧视频使用 `object-fit: cover` 裁切进品牌视觉卡片。小于 768px 时改为单列，卡片宽度 100%，仍保持圆角和正方形比例。

## 10. 当前第一版范围

第一版必须完成：

- 登录和角色权限。
- 编导、剪辑、管理员核心工作台。
- 脚本库、新增脚本、脚本详情和编辑。
- 视频任务下达、自动分配剪辑、任务详情。
- 剪辑状态流转。
- 审核建议、审核历史和截图附件。
- 可发布、屯片、排期、手动发布、库存归档。
- 人员和目标管理。
- 个人头像和配色设置。
- 16:30 站内提醒。

第一版暂不做：

- 外部平台自动发布。
- 成片视频文件上传。
- 多组织或多团队租户。
- 复杂报表导出。

## 11. 验收标准

- 不同角色登录后只能看到和操作自己权限内的数据。
- 编导今日完成数能按当天脚本数据正确计算。
- 剪辑今日完成数能按当天交付数据正确计算。
- 新增多条视频任务时，剪辑人按未完成任务最少分配。
- 审核闭环可以完成待审核 -> 需修改 -> 待审核 -> 审核通过，并保留历史。
- 无发布日期的视频进入屯片池，有发布日期的视频进入排期或可发布。
- 手动发布后进入已发布库存。
- 头像和审核截图可上传、预览，刷新后仍可访问。
- 16:30 后未完成用户出现站内提醒。
- 桌面和移动端核心页面无明显溢出，浏览器控制台无本地页面错误。
